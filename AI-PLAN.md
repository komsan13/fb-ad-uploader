- ทดสอบใน testMode ก่อนเสมอ, เทสทั้งชุดต้องผ่านก่อน commit
- ค่าใช้จ่าย API ต่ำมากเทียบงบแอด (หลักสิบ–ร้อยบาท/เดือน)

## เฟส 1.5: โหมด AI เต็มระบบ (เพิ่มตามคำขอ 20 ก.ค. 2026)

- [x] **สวิตช์เดียว "โหมด AI เต็มระบบ" (`autopilot.aiFull`)** ในการ์ด Autopilot บนแดชบอร์ด — เปิดแล้วบังคับเปิดระบบอัตโนมัติด้วย และเพิ่ม 2 พฤติกรรม:
  - ตรวจนโยบายคู่ วิดีโอ×แคปชั่น ก่อนเติมแอดทุกตัว (cache ผลต่อคู่ 7 วันใน `pfCache`) — เสี่ยงสูง = ข้ามคู่นั้น, AI ล่ม = ปล่อยผ่านแบบเดิม (fail-open)
  - คลังแคปชั่นใกล้หมด → AI เขียนแบบร่างเอง (วันละครั้ง) แล้วส่ง**ข้อความเต็ม**เข้า Telegram ให้พิมพ์ "ยืนยัน" ครั้งเดียวเพื่ออนุมัติทั้งชุด — ผู้ใช้เลือกแบบยืนยันผ่าน Telegram (ไม่อนุมัติอัตโนมัติ)
  - จงใจไม่ให้ AI ในแชทเสนอ `approveCaptions` เอง — ข้อเสนอมาจากโค้ดที่แนบข้อความเต็มเท่านั้น
  - เกราะจากรอบรีวิว: ข้อเสนอไม่ทับ pending อื่น + arm หลังส่งถึงจริงเท่านั้น, แก้/ลบแคปชั่นแล้วล้าง `pfCache` ทันที, AI ปฏิเสธการตรวจ = ถือว่าเสี่ยงสูง, ตัวตรวจล่มมีแจ้งเตือนวันละครั้ง (fail-open แบบมีร่องรอย), โควตาตรวจสด 12 ครั้ง/รอบกันรอบตรวจลากยาวจนเลนจับแอดโดนปฏิเสธอดวิ่ง, สวิตช์เช็ค key/Telegram ก่อนเปิดและ confirm ก่อนพ่วงเปิดระบบใช้เงินจริง

## เฟส 2: AI เข้าใกล้การตัดสินใจมากขึ้น (เสร็จ 20 ก.ค. 2026)

- [x] **2.4 จับคู่วิดีโอ×แคปชั่นด้วยข้อมูล** — จด `adMeta` (แอด→วิดีโอ/แคปชั่น/สกุลเงิน) ตอนเติมแอด, snapshot `adPerf` 3 วันจาก insights ที่ตัวปิดแอดขาดทุนดึงอยู่แล้ว (ฟรี), ตอน refill จัดอันดับ: ตัวชนะ (CPA ≤ เป้า) ขึ้นก่อน, ยังไม่รู้หมุนตามคิว, ตัวที่พิสูจน์ว่าแพ้ (เกณฑ์เดียวกับตัวปิดแอด) ถูกตัดออก — สถิติแยกต่อสกุลเงินเสมอ
- [x] **2.5 AI เสนอปรับเป้า CPA (`ruleCpr`)** — ทุกวันจันทร์ 08:00: อ่านผล 7 วันต่อบัญชี เสนอค่าใหม่ผ่าน Telegram ให้พิมพ์ "ยืนยัน" (เกราะฝั่งเรา: ปัดตกข้อเสนอกระโดดเกิน ±50%) + สั่งเองผ่านแชทได้ (`setRuleCpr`)
- [x] **2.6 AI ตรวจสุขภาพหน้า Landing** — ทุกวันพุธ: ยิงลิงก์จริงหา 404 + Claude เทียบความสอดคล้อง landing กับแคปชั่น แจ้งเตือนเฉพาะเมื่อมีปัญหา
